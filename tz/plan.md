## Роли

В системе 3 роли:

- **admin** — полный доступ ко всему: контент, пользователи, модерация, справочники, журнал действий, системные настройки
- **moderator** — модерация комментариев, управление объектами и медиа (без прав на пользователей и системные настройки)
- **user** — личный кабинет, избранное, комментарии, оценки, маршруты

Роли хранятся в таблице `user_roles` (ASP.NET Identity). Связь пользователя с ролями — через `user_role_map`. Сидятся автоматически в `DbSeeder` при первом запуске. `DbSeeder` также мигрирует существующих пользователей со старых ролей `superadmin → admin` и `editor → user`, после чего удаляет устаревшие роли (идемпотентно).

## База данных

Полная схема в `schema.sql`. Краткая выжимка:

**Справочники:** `regions`, `categories`, `tags`, `arch_styles`

**Объект и всё вокруг него:**
- `heritage_objects` — главная таблица. FULLTEXT индексы на названиях и описаниях для поиска
- `object_images` — галерея
- `object_locations` — координаты (lat/lng)
- `object_tag_map` — связь многие-ко-многим с тегами
- `object_relations` — связанные объекты

**Юзеры (ASP.NET Identity с кастомным маппингом имён таблиц):**
- `users` — расширенная `IdentityUser<int>`: email, username, normalized_username/email, password_hash (PBKDF2 от Identity), security_stamp, concurrency_stamp, phone_number/confirmed, two_factor_enabled, lockout_end/enabled, access_failed_count, email_confirmed + кастомные поля (display_name, avatar_url, bio, preferred_lang, is_active, google_id, role, last_login_at)
- `user_roles` — `IdentityRole<int>` (name, normalized_name, concurrency_stamp)
- `user_role_map` — связь многие-ко-многим пользователь↔роль (IdentityUserRole)
- `user_claims` — IdentityUserClaim (claim_type / claim_value на пользователя)
- `user_logins` — IdentityUserLogin (внешние провайдеры логина: Google и т.п.)
- `role_claims` — IdentityRoleClaim (claim'ы на роль)
- `asp_user_tokens` — IdentityUserToken (внутренние Identity-токены: 2FA, refresh и т.п.)
- `user_tokens` — **кастомная** таблица токенов приложения: email-верификация, сброс пароля, собственный refresh; не пересекается с `asp_user_tokens`

**Активность:**
- `favorites` — избранное
- `comments` — комментарии со статусом модерации
- `ratings` — оценки 1–5 (3 триггера автоматически обновляют `rating_avg` в `heritage_objects`)
- `routes` + `route_stops` — маршруты

**Контент:**
- `quizzes`, `quiz_questions`, `quiz_answers`, `quiz_results`
- `ornaments` — сохранённые орнаменты
- `timeline_events` — события таймлайна (управляются через админку)
- `audit_log` — лог действий в админке с JSON-полем для diff

## Страницы

### Главная `/`

**Hero-блок**
Полноэкранный с фоновым изображением/видео (параллакс или орнамент/замок). Крупный заголовок «Культурное наследие Беларуси», подзаголовок на трёх языках, две CTA-кнопки: «Каталог» и «Карта». Анимация появления через AOS.

**Счётчики**
Анимированная строка цифр: всего объектов, регионов охвата, фотографий в галерее, зарегистрированных пользователей. Цифры считаются от нуля при попадании в viewport (Intersection Observer + GSAP).

**Избранные объекты**
Swiper.js карусель из 6–8 объектов с флагом `is_featured = true`. Каждая карточка: главное фото, название, регион, категориальная иконка, рейтинг. Стрелки навигации, точки-пагинация, автоплей.

**Категории**
Иконочная сетка 2×5 (или адаптивно). Клик по категории → `/catalog?category=slug`. У каждой ячейки: SVG-иконка, название, число объектов в категории.

**Сегодня в истории**
Блок с объектом или событием, дата которого совпадает с текущим числом/месяцем. Данные берутся из `timeline_events`. Если совпадений нет — скрывается.

**Превью таймлайна**
Горизонтальный скролл с несколькими ближайшими эпохальными событиями, ссылка «Смотреть всю историю» → `/timeline`.

**Квизы-тизер**
3 карточки с кратким описанием квиза, превью-изображением и кнопкой «Пройти». Ссылка «Все квизы» → `/quizzes`.

**Баннер конструктора орнамента**
Секция с анимированным SVG-орнаментом, кратким описанием и кнопкой «Создать орнамент».

**Footer**
Логотип, навигационные колонки (разделы сайта), переключатель языка (ru / be / en), копирайт, ссылка на GitHub (если есть).

### Каталог `/catalog`

**Фильтры (боковая панель или панель сверху):**
- Регион — чекбоксы (7 регионов из БД)
- Категория — чекбоксы с иконками (из БД)
- Век — range-слайдер (XII–XXI)
- Теги — мультивыбор с поиском по тегам
- Статус сохранности — `preserved / partial / ruins / lost` (радиокнопки или чекбоксы)
- Наличие фото — переключатель да/нет
- Посещаемость — переключатель «только доступные для посещения»

Все фильтры применяются через AJAX без перезагрузки страницы. Кнопка «Сбросить фильтры». URL обновляется через `history.pushState`, чтобы ссылку можно было скопировать.

**Переключатель отображения:**
Сетка (2–3 колонки) / список (строки). Настройка сохраняется в `localStorage`.

**Сортировка:**
По алфавиту, по дате добавления, по рейтингу, по веку постройки.

**Пагинация:**
12 объектов на страницу. Стандартная постраничная пагинация с указанием текущей страницы и общего числа. URL: `/catalog?page=2&...filters`.

**Карточка объекта в каталоге:**
Главное фото (aspect-ratio 4:3), название, регион + иконка, категория + иконка, короткое описание (≤120 символов, обрезается с «…»), рейтинг звёздочками + число оценок, кнопка «В избранное» (сердечко). Клик по карточке → страница объекта.

**Шапка результатов:**
«Найдено N объектов», текущий набор активных фильтров в виде бейджей с крестиком (для быстрого удаления каждого фильтра).

### Карточка объекта `/object/{slug}`

**Шапка страницы:**
Хлебные крошки: Главная → Каталог → [Категория] → Название объекта. Переключатель языка для переключения отображаемого контента (ru/be/en).

**Галерея:**
Swiper.js слайдер с миниатюрами под основным изображением. Клик → лайтбокс Lightgallery. 360°-панорамы отмечены иконкой и открываются в Pannellum. Подписи к фотографиям под слайдером.

**Основная информация (левая/главная колонка):**
- Название на текущем языке (H1), иконка категории рядом
- Регион и населённый пункт
- Век постройки / основания (диапазон, например «XIII–XV вв.»)
- Архитектурный стиль
- Статус сохранности — цветной бейдж (сохранился / частично / руины / утрачен)
- Охранный статус по закону РБ (категория 1/2/3 + год внесения в реестр)
- Время работы и стоимость входа (если объект посещаемый)
- Полное описание с форматированием (WYSIWYG-контент из `description_ru/be/en`)

**Боковая панель (правая колонка):**
- Мини-карта Leaflet с маркером. Кнопка «Показать на большой карте» → `/map?focus={id}`. Кнопка «Добавить в маршрут».
- Координаты и адрес
- Рейтинг — 5 кликабельных звёзд, среднее значение + число оценок. Авторизованный пользователь ставит/меняет свою оценку через AJAX. Неавторизованному — тултип «Войдите, чтобы оценить».
- Кнопка «В избранное» (сердечко, AJAX)
- Кнопка «Сообщить об ошибке» — модальное окно с полями «Тема» и «Описание», отправляется в `audit_log`

**Дополнительные поля:**
- Теги — кликабельные цветные бейджики → `/catalog?tag=slug`
- Архитектор / строитель
- Площадь (га), если указана
- Охранный статус + год внесения
- Интересный факт — выделенный блок с иконкой 💡
- Связанные объекты — горизонтальный ряд из 3–5 карточек-миниатюр (slug → Detail)
- Ближайшие объекты — топ-5 в радиусе ~30 км (Haversine в SQL), такой же ряд карточек

> **Убрано из доп. полей:** заказчик/исторический владелец (`commissioner`), материал строительства (`material`). Эти поля остаются в БД и форме редактирования, но **не отображаются** на публичной странице объекта.

**Комментарии:**
Блок внизу страницы. Отображаются только одобренные комментарии. Пагинация: **10 комментариев на страницу**, постраничная навигация (без бесконечной прокрутки). Каждый комментарий: аватар, имя пользователя, дата, текст. Форма добавления — только для авторизованных пользователей; после отправки показывается плашка «Комментарий отправлен на модерацию». Неавторизованным — блок «Войдите, чтобы оставить комментарий».

### Поиск

**Строка поиска в шапке:**
Всегда доступна. jQuery UI Autocomplete — AJAX на `/search/autocomplete?q=` при вводе от 2 символов. API возвращает топ-10: название + регион + иконка категории. Нажатие Enter или кнопки → полная страница `/search?q=...`.

**Полная страница `/search?q=...`:**
Заголовок «Результаты поиска по: „запрос"». Количество найденных объектов. Список карточек с подсветкой совпадений (теги `<mark>`). Фильтры аналогичны каталогу (регион, категория, век). Пагинация 12 объектов. Поиск ведётся по названию (FULLTEXT: `name_ru/be/en`, `short_desc_ru`), тегам, категории через `MATCH ... AGAINST`.

### Карта `/map`

Карта — **полноценная отдельная страница** (не встроена в каталог, не полноэкранная). Страница содержит:

**Шапка страницы:**
Навигационная панель с фильтрами и поиском в одну строку над картой. Хлебные крошки: Главная → Карта.

**Панель фильтров и поиска (над картой или в сайдбаре):**
- Строка поиска с автодополнением (фильтрует/смещает карту к найденному объекту)
- Категория — чекбоксы с цветными иконками
- Регион — выпадающий список или чекбоксы
- Статус сохранности — мультивыбор
- Теги — мультивыбор
- Переключатель «Только посещаемые»
- Кнопка «Сбросить»
- Кнопка «Моё местоположение» (Geolocation API)

При изменении фильтров маркеры обновляются через AJAX без перезагрузки страницы.

**Карта (основная область):**
Leaflet.js + MapLibre GL-слой (локализованный векторный стиль через OpenFreeMap), с fallback на OSM raster tiles при недоступности style/maplibre. Занимает всё оставшееся пространство страницы (`calc(100vh - header)`). Кастомные SVG-маркеры по категории (цвет и иконка из БД), динамическая перерисовка по фильтрам без перезагрузки.

**Попап при клике на маркер:**
Миниатюра, название, регион, категория, краткое описание (до 80 символов), кнопка «Подробнее» → `/object/{slug}`, кнопка «В маршрут».

**Боковая панель результатов (опционально, по умолчанию скрыта):**
Список объектов в текущем видимом bbox карты. Скролл-синхронизация: при наведении на строку список подсвечивает маркер на карте и наоборот.

> **Полноэкранный режим не предусмотрен.** Страница `/map` сама по себе является полноценным видом — с хедером, фильтрами и картой во всю высоту viewport.

### Таймлайн `/timeline`

**Концепция:**
Вертикальная лента событий, где **отображаются только те даты и годы, для которых в БД (`timeline_events`) существуют опубликованные события**. Никаких «пустых» столетий или лет — таймлайн полностью динамический и управляется через админку.

**Структура ленты:**
- Группировка по **году** (заголовок года: `1387`, `1793`, `1941`…). Внутри года — по конкретным датам если они указаны.
- Если в `timeline_events` нет ни одного события за данный год — год не рендерится вообще.
- Эпохальные «разделители» (ВКЛ, Речь Посполита, Российская Империя, БССР, Республика Беларусь) вставляются только там, где соответствующий период начинается среди имеющихся событий.

**Карточка события:**
Год + дата (если указана), заголовок, краткое описание (`body_ru/en`), миниатюра (если есть `image_url`). Если событие привязано к объекту (`object_id IS NOT NULL`) — кнопка «Перейти к объекту» → `/object/{slug}`.

**Вертикальная линия:**
SVG-линия прорисовывается через AOS-анимацию при скролле. Маркеры-точки на линии для каждого события.

**Фильтрация:**

**Управление через админку:**
Все события создаются/редактируются/удаляются в разделе `/admin/timeline`. Поле `is_published = false` — событие скрыто с публичного таймлайна. Поле `year` — обязательное, `sort_order` — для сортировки событий внутри одного года.

### Квизы `/quizzes`

**Список квизов:**
Карточки с заголовком, описанием, количеством вопросов, сложностью (если есть) и кнопкой «Пройти». Пять типов:

### Конструктор маршрута `/route-builder`

**Добавление объектов:**

### Конструктор орнамента `/ornament-builder`

Canvas-приложение на нативном Canvas API.

- Белое полотно (лён) как фон
- **Режим «Кусочки»**: библиотека традиционных ромбовидных/крестообразных/растительных элементов. Клик на элемент → клик на холст → размещается. Перетаскивание, отражение по горизонтали/вертикали
- **Режим «Кисть»**: свободное пиксельное рисование с выбором цвета и размера кисти

- Скачать PNG — `canvas.toDataURL()` → `<a download>`
- Авторизованный пользователь может сохранить и опубликовать → галерея `/ornaments`

## Важные моменты

- **Загрузка файлов** — сохранять в `wwwroot/uploads/objects/{objectId}/`, имя файла = GUID, ресайзить через ImageSharp до максимум 1920px и отдельно thumb 400px
- **Slug** — генерировать автоматически из названия (ru → транслит) при создании объекта, хранить в БД, не менять после публикации
- **Soft delete** — у объектов всегда `is_deleted`, физически не удалять; восстановление через `/admin/trash`
- **Корзина** — при восстановлении объект получает статус `draft` и логируется в `audit_log`
- **Таймлайн** — рендерить только те годы/даты, для которых есть записи с `is_published = true` в `timeline_events`; запрос: `SELECT DISTINCT year FROM timeline_events WHERE is_published = 1 ORDER BY year`
- **Пагинация комментариев** — 10 записей на страницу, стандартная постраничная навигация
- **Карта `/map`** — полноценная страница с хедером + фильтры + Leaflet на calc(100vh - header); полноэкранный режим не предусмотрен
- **Карта `/map` (реализовано)** — базовый слой приведён к тому же подходу, что и в `/RouteBuilder` и `/RouteBuilder/Share`: `MapLibre + localized style + OSM fallback`
- **Admin Dashboard (реализовано)** — добавлен переключатель языка `ru/be/en`
- **Поля `commissioner` и `material`** — остаются в БД и схеме, но не отображаются в публичном UI (ни на карточке объекта, ни в форме редактирования)
- **Триггеры рейтинга** — уже в `schema.sql`, пересчитывают `rating_avg` и `rating_count` автоматически
- **JWT** — хранить в httpOnly Cookie (`auth_token`), не в localStorage
- **ASP.NET Identity + кастомный маппинг** — `User : IdentityUser<int>`, `UserRole : IdentityRole<int>`. Все Identity-сущности явно замаплены в `AppDbContext.OnModelCreating` на кастомные snake_case-имена таблиц (`users`, `user_roles`, `user_role_map`, `user_claims`, `user_logins`, `role_claims`, `asp_user_tokens`). Дубли с Identity (`Username`, `EmailVerified`) в `User` убраны — используются унаследованные `UserName` / `EmailConfirmed`.
- **Сидер** — `Data/DbSeeder.cs`: при старте приложения создаёт все 5 ролей и дефолтного админа через `RoleManager`/`UserManager`. Вызов из `Program.cs` после `app.Build()`.
- **Кодировка БД** — `utf8mb4` обязательна. Раскатывать миграцию `mysql --default-character-set=utf8mb4 < 001_InitialSchema.sql` (иначе на Windows mysql.exe прочитает файл в cp1251 и запишет mojibake). Connection string содержит `CharSet=utf8mb4`. В SQL-файле есть `SET NAMES utf8mb4` на случай запуска без флага.
- **Переключение языка** — через кнопки в `_Header.cshtml` → `HomeController.SetLanguage(culture)` ставит cookie `culture` (`ru`/`be`/`en`). Нормализация кода языка и UI-переводы выполняются централизованно через `Localization/UiText.cs`.
- **Комментарии** — по умолчанию статус `pending`, показывать только `approved`
- **FULLTEXT поиск** — индексы уже в `schema.sql` на `name_ru`, `short_desc_ru`, `description_ru`
- **Ближайшие объекты** в карточке — считать через Haversine формулу в SQL:
  ```sql
  ORDER BY (6371 * ACOS(COS(RADIANS(:lat)) * COS(RADIANS(lat)) 
    * COS(RADIANS(lng) - RADIANS(:lng)) 
    + SIN(RADIANS(:lat)) * SIN(RADIANS(lat))))
  LIMIT 5
  ```
